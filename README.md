# Arbeiten

A local-first, Todoist-style desktop todo app for macOS, built with Electron. All
data lives in a local SQLite database — no accounts, no cloud.

![icon](build/icon.png)

## Features

- **Tasks** with projects, labels, priorities (P1–P4), due dates, descriptions,
  sub-tasks, comments, and an estimated **duration** badge.
- **Quick Add** with natural-language tokens: `Email boss tomorrow p1 #Work @urgent`.
- **Views**: Today, Upcoming (grouped by date), Inbox, per-Project, per-Label,
  search, and priority filters.
- **Drag-and-drop** reordering of tasks.
- **Focus panel**: a Timer (stopwatch / countdown) and a Pomodoro mode (25/5, long
  break every 4th cycle). Both fire a native notification + chime on completion.
- **Recurring tasks** (daily / weekdays / weekly / monthly / every-N-days).
- **Productivity** stats: completed today, last 7 days, current streak, daily goal.
- **Light & dark** themes (persisted).

## Tech

Electron · electron-vite · React 18 + TypeScript · Zustand · better-sqlite3 ·
dnd-kit · electron-builder. The renderer talks to SQLite only through a typed
`window.api` IPC bridge (`contextIsolation` on, `nodeIntegration` off).

## Develop

```bash
npm install
npm run dev        # launch with hot reload
npm test           # vitest (parser, recurrence, time, migrations, repositories)
npm run typecheck
```

`npm run dev`/`test` automatically rebuild the native `better-sqlite3` module for the
correct ABI (Electron for dev, Node for tests).

## Build the installable .dmg

```bash
npm run dist
```

Produces `dist/Arbeiten-<version>.dmg`. Open it, drag **Arbeiten** to Applications,
and launch it from Launchpad/Dock. The "Ar" app icon is generated by
`scripts/make-icon.mjs` (`npm run make-icon`).

The database is stored at
`~/Library/Application Support/Arbeiten/arbeiten.db`.

## Agent integration

An agent (Claude Code or any MCP client) can add and manage tasks/labels. One-time build:

```bash
npm run agent:setup
```

- **Skill** — `.claude/skills/arbeiten/` lets Claude Code drive the `agent/` CLI
  (`node agent/dist/cli.js add-task --content "..." --project Work --due tomorrow --labels work`).
- **MCP server** — `.mcp.json` registers an `arbeiten` server exposing `add_task`,
  `add_label`, `complete_task`, `update_task`, `list_tasks`, etc.

Both write directly to the SQLite DB (works whether or not the app is open) and reuse
the app's data layer. See `agent/README.md` for the why/how.

## Layout

```
src/shared/     domain + API types, pure logic (quick-add parser, recurrence, time)
src/main/       app lifecycle, SQLite (db/), IPC handlers, notifications
src/preload/    typed window.api contextBridge
src/renderer/   React UI (components/, views, Zustand store, styles)
```

See `docs/superpowers/specs/` and `docs/superpowers/plans/` for the design + plan.
